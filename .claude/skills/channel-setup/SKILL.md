# Channel Setup Skill

유튜브 채널 설정을 대화형으로 생성하는 스킬.

## 역할 원칙

1. **추천 먼저, 선택은 사용자**: 모든 질문에 추천 선택지를 먼저 제시. 주관식 최소화.
2. **단계적 진행**: 한 번에 많이 묻지 않고, 2~3개씩 끊어서 질문.
3. **벤치마킹 활용**: URL이 있으면 분석 결과를 기반으로 더 정교한 추천 제시.
4. **최소 질문**: 벤치마킹으로 추론 가능한 건 직접 묻지 않고 추천으로 대체.

---

## 전체 흐름

```
STEP 1: 장르 선택
STEP 2: 벤치마킹 영상 URL (선택사항)
  → URL 있으면: collect.py로 수집 → 대본 분석 → 추천 근거 확보
STEP 3: 채널 성격 질문 (벤치마킹 유무에 따라 질문 방식 달라짐)
  → 라운드 1~3: 톤, 표현, 관점
  → 라운드 4~8: 썸네일 전략
STEP 4: profile.md + settings.json + workflow.json + thumbnail-strategy.json 자동 생성
```

---

## STEP 1: 장르 선택

아래 표를 보여주고 선택받는다. 표에 없는 장르도 직접 입력 가능.

```
어떤 장르의 채널을 만들까요?

| 번호 | 장르        | 이런 영상을 만듭니다                          |
|------|-------------|----------------------------------------------|
| A    | 경제/시사   | 삼성 위기, 부동산 폭락, 기업 분석             |
| B    | 역사        | 조선시대 미스터리, 전쟁 비화, 인물 이야기     |
| C    | 심리학      | 왜 충동구매를 할까, 첫인상의 비밀, 습관의 과학 |
| D    | 과학/기술   | 블랙홀의 비밀, AI의 미래, 양자역학 쉽게       |
| E    | 건강/의학   | 수면의 과학, 식습관의 진실, 운동의 효과        |
| F    | 직접 입력   | 원하는 장르를 자유롭게                        |
```

→ 선택된 장르를 `{장르}`에 저장

---

## STEP 2: 벤치마킹 영상 (선택사항)

```
"이런 느낌으로 만들고 싶다" 싶은 유튜브 영상이 있나요?
URL을 1~3개 붙여주시면, 분석해서 채널 설정에 반영합니다.
없으면 그냥 엔터 치세요 — 없어도 괜찮습니다!
```

### URL을 받은 경우 (벤치마킹 있음)

1. 채널 폴더 생성: `channels/{channel-id}/config/_benchmarks/`
2. collect.py로 수집:
   ```bash
   .venv/bin/python scripts/collect.py --project _benchmarks --channel "{channel-id}" URL1 URL2 ...
   ```
   **주의**: collect.py는 `projects/` 하위에 저장하므로, 수집 후 `_benchmarks/` 폴더를 `config/` 아래로 이동한다:
   ```bash
   mv channels/{channel-id}/projects/_benchmarks/_refs/* channels/{channel-id}/config/_benchmarks/
   rm -r channels/{channel-id}/projects/_benchmarks
   ```
3. 각 `_benchmarks/{NNN}/transcript.txt`와 `meta.md`를 Read
4. **분석 수행** — 아래 항목을 추출:
   - 톤/말투 (존댓말? 반말? 친근? 권위적?)
   - 서사 구조 (시간순? 나열형? 반전?)
   - 정보 전달 방식 (데이터? 사례? 비유?)
   - 자주 쓰는 표현 / 안 쓰는 표현
   - 관점 (소비자? 전문가? 일반인?)
   - 영상 평균 길이
5. 분석 결과를 `{벤치마크_분석}`에 저장 → STEP 3에서 추천 근거로 활용

### URL을 안 받은 경우 (벤치마킹 없음)

→ 장르 기반 기본 추천으로 STEP 3 진행

## STEP 3: 채널 성격 질문

**벤치마킹 유무에 따라 질문 방식이 달라진다.**

```

- 톤: 친한 형이 설명해주는 느낌, 반말 + 존댓말 혼용
- 서사: 주로 몰락 서사, 시간순 전개
- 전달 방식: 숫자 데이터를 일상 비유로 풀어줌
- 특징적 표현: "자, 여기서 핵심인데요", "숫자로 보면요"
- 평균 영상 길이: 12분

이 스타일을 기반으로 추천드릴게요.

Q1. 채널 소개 한 문장 — 시청자가 "이 채널은 _____ 채널이야"라고 말하듯:
  → 추천: "{분석 기반 추천 문장}"
  그대로 쓸까요, 수정할까요?

Q2. 말투는 이 느낌이 맞나요?
  → 추천: {분석에서 추출한 톤}
  다른 걸로 바꾸고 싶으면 골라주세요:
  A. 친한 형/언니가 설명해주는 느낌
  B. 다큐멘터리 내레이터
  C. TED 강연자처럼 설득력 있게
  D. 밤에 친구한테 이야기 들려주듯

#### 벤치마킹 없을 때

장르 기반 기본 추천:

```

  장르별 예시:
  - 경제: "경제 뉴스를 직장인 언어로 번역해주는 채널"
  - 역사: "역사 속 소름끼치는 이야기를 들려주는 채널"
  - 심리학: "일상에 숨은 심리학을 발견하는 채널"
  - 과학: "어려운 과학을 5분 만에 이해시켜주는 채널"

  A. 친한 형/언니가 설명해주는 느낌 ← {장르}에 추천

  C. TED 강연자처럼 설득력 있게
  D. 밤에 친구한테 이야기 들려주듯
  E. 직접 입력
```

**장르별 기본 추천 톤:**
- 경제/시사 → A (친한 형)
- 역사 → D (밤에 친구한테)
- 심리학 → C (TED 강연자)
- 과학/기술 → A (친한 형)
- 건강/의학 → C (TED 강연자)

### 서사 구조는 직접 묻지 않는다

- **벤치마킹 있으면**: 분석에서 추출한 서사 구조를 profile.md에 자동 반영
- **벤치마킹 없으면**: 장르 기반 기본 서사 구조를 자동 설정

**장르별 기본 서사 구조:**
| 장르 | 기본 선호 | 기본 기피 |
|------|-----------|-----------|
| 경제/시사 | 몰락 서사, 반전 서사 | — |
| 역사 | 연대기 서사, 폭로 서사 | 나열형 |
| 심리학 | 반전 서사, 폭로 서사 | 연대기 서사 |
| 과학/기술 | 반전 서사, 폭로 서사 | — |
| 건강/의학 | 반전 서사, 나열형 | — |

→ profile.md 생성 시 자동 포함. 사용자가 나중에 수정 가능하다고 안내.

### 라운드 4: 썸네일 스타일 & 장수

```
이제 썸네일 전략을 설정할게요. 매 영상마다 AI가 썸네일 이미지를 자동 생성하는데,
채널 스타일에 맞게 설정해두면 일관된 느낌의 썸네일이 나옵니다.

Q7. 썸네일 몇 장 만들까요?
  A. 3장 (빠르게 골라서 사용)
  B. 5장 (다양한 선택지) ← 추천
  C. 7장 (최대한 많이 보고 고르기)

Q8. 선호하는 아트 스타일은? (복수 선택 가능)
  A. 실사/시네마틱 — 영화 포스터, 뉴스 보도 느낌
  B. 애니메/일러스트 — 만화, 웹툰 느낌
  C. 직접 입력 — 원하는 스타일을 자유롭게 (예: "3D 렌더링", "수채화풍", "픽셀아트")
```

**장수별 배분 공식:**
- 스타일 2개 + 장수 5: 컨셉A(스타일1, 스타일2) + 컨셉B(스타일1, 스타일2) + 와일드카드 1장
- 스타일 1개 + 장수 3: 컨셉A(1장) + 컨셉B(1장) + 와일드카드 1장
- 스타일 2개 + 장수 7: 컨셉A(스타일1, 스타일2) + 컨셉B(스타일1, 스타일2) + 와일드카드 3장
- 일반 공식: 와일드카드 1장 확보 → 나머지를 컨셉A/B × 스타일 수로 균등 배분

**장르별 기본 추천:**
- 경제/시사 → A+B (실사+애니메)
- 역사 → A (실사/시네마틱)
- 심리학 → A+B (실사+애니메)
- 과학/기술 → A+B (실사+애니메)
- 건강/의학 → A (실사/시네마틱)

### 라운드 5: 색감 & 감정톤

```
Q9. 전체적인 색감/분위기는?

| 선택 | 색감                                | 잘 어울리는 장르                      |
|------|-------------------------------------|--------------------------------------|
| A    | 밝고 선명 (고채도, 화이트 배경)       | 교육, 리뷰, 자기계발, 요리            |
| B    | 다크/시네마틱 (어두운 배경, 강한 대비) | 경제위기, 미스터리, 범죄, 공포         |
| C    | 파스텔/소프트 (부드러운 톤)           | 심리, 힐링, 육아, 일상 브이로그        |
| D    | 네온/사이버 (형광, 글로우)            | 테크, AI, 게임, SF                   |
| E    | 모노톤 + 포인트컬러 (흑백 + 빨강 등)  | 다큐, 역사, 시사, 고급 브랜드         |
| F    | 주제에 따라 자동 결정                 | 장르가 다양하거나 매번 다르게 하고 싶을 때 |

  → 추천: {장르 기반 또는 벤치마크 분석 기반}

Q10. 썸네일에서 유발하고 싶은 감정은? (1~2개 선택)

| 선택 | 감정          | 표현 방식                                    |
|------|--------------|---------------------------------------------|
| A    | 충격/놀라움    | 과장된 표정, 강렬한 색대비, "이게 진짜?"        |
| B    | 호기심/궁금증  | 일부만 보여주기, 의문형 구도, "뭐지?"           |
| C    | 위기/긴장감    | 균열, 하락, 어두운 톤, "큰일났다"              |
| D    | 신뢰/전문성    | 깔끔한 레이아웃, 데이터 시각화, 차분한 느낌      |
| E    | 따뜻함/공감    | 부드러운 색감, 인물 중심, 일상적 장면            |
| F    | 유머/재미      | 밝은 색, 코믹한 표현, 과장된 상황               |
| G    | 고급/프리미엄  | 미니멀, 블랙&골드, 세련된 느낌                  |

  → 추천: {장르 기반 또는 벤치마크 분석 기반}
```

**장르별 기본 추천 색감:**
- 경제/시사 → B (다크/시네마틱)
- 역사 → E (모노톤 + 포인트컬러)
- 심리학 → C (파스텔/소프트)
- 과학/기술 → D (네온/사이버)
- 건강/의학 → A (밝고 선명)

**장르별 기본 추천 감정:**
- 경제/시사 → A+C (충격 + 긴장)
- 역사 → B+C (호기심 + 긴장)
- 심리학 → B+E (호기심 + 따뜻함)
- 과학/기술 → A+B (충격 + 호기심)
- 건강/의학 → D+E (신뢰 + 따뜻함)

  C. 좌우 분리 — 한쪽 이미지 / 반대쪽 텍스트 공간

Q12. 인물(사람 얼굴) 표현은?
  A. 과장된 표정 클로즈업 — 놀란 얼굴, 충격 표정 (유튜브 클래식)
  B. 자연스러운 인물 — 실제 인물 사실적 표현
  C. 실루엣/뒷모습 — 미스터리한 느낌, 얼굴 안 보임
  D. 인물 최소화 — 오브젝트/그래픽 중심, 사람 거의 안 넣음
  E. 주제에 따라 자동 — 인물이 중요한 주제만 넣음
```

### 라운드 7: 브랜드 일관성

```
Q13. 매 영상 썸네일에 반복되는 브랜드 요소를 넣을까요?
  채널 인지도를 높이는 데 효과적입니다. (복수 선택 가능)

  A. 브랜드 컬러 — 매 썸네일에 반드시 들어갈 색상
     → "어떤 색인가요?" (1~3개, 예: "#FF0000", "민트색", "네이비+골드")
  B. 마스코트/캐릭터 — 채널 캐릭터가 매번 등장
     → assets/mascot/ 폴더에 이미지 1장 추가 안내
  C. 고정 레이아웃 — 항상 같은 구도 패턴
     → "어떤 구도?" (예: "항상 좌측에 인물, 우측에 핵심 오브젝트")
  D. 없음 — 매번 자유롭게
```

### 라운드 8: 참조 자료 (선택사항)

```
Q14. 채널 마스코트나 이미지 스타일 레퍼런스가 있나요?

  마스코트 이미지나 "이런 스타일로 그려줘" 레퍼런스가 있으면
  썸네일 생성 시 참고합니다.

  → 있으면: 아래 폴더에 이미지를 1장씩 넣어주세요.
    - 마스코트: channels/{channel-id}/config/assets/mascot/
    - 스타일 레퍼런스: channels/{channel-id}/config/assets/style-reference/

  ⚠️ 참고: AI 이미지 생성에서 마스코트 일관성은 제한적입니다.
  정확히 같은 캐릭터가 나오지 않을 수 있으며, 참고용으로 활용됩니다.

  → 없으면: 건너뛰기 (나중에 언제든 추가 가능)

Q15. 참고하고 싶은 유튜브 썸네일이 있나요? (선택사항)

  "이런 썸네일 느낌이 좋다" 싶은 유튜브 영상 URL을 1~3개 붙여주세요.
  썸네일을 분석해서 스타일에 반영합니다.
  없으면 그냥 엔터 — 위에서 고른 설정으로 충분합니다!
```

#### 벤치마크 썸네일 분석 (Q15에서 URL을 받은 경우)

1. YouTube URL에서 VIDEO_ID 추출
2. 썸네일 다운로드: `https://img.youtube.com/vi/{VIDEO_ID}/maxresdefault.jpg`
   - maxresdefault 실패 시 `hqdefault.jpg`로 폴백
3. 다운로드한 이미지를 Read로 직접 분석 (Claude 멀티모달):
   - 구도 패턴 (상하분리? 좌우? 중앙?)
   - 색감/톤 (밝은? 어두운? 네온?)
   - 아트 스타일 (실사? 일러스트? 3D?)
   - 감정 톤 (충격? 호기심? 차분?)
   - 인물 표현 (과장? 자연? 없음?)
   - 텍스트 배치 (어디에? 몇 줄?)
4. 분석 결과 요약을 `thumbnail-strategy.json`의 `benchmark_analysis`에 저장
5. 앞서 사용자가 선택한 설정과 충돌하면, 분석 결과를 참고 정보로만 활용 (사용자 선택 우선)

## STEP 4: 채널 생성

### 채널 폴더 이름 결정

장르 기반으로 영어 kebab-case 자동 생성:
- 경제/시사 → `economy`
- 역사 → `history`
- 심리학 → `psychology`
- 과학/기술 → `science`
- 건강/의학 → `health`
- 기타 → 장르에서 영어 키워드 추출

### 파일 생성

#### 1. `channels/{channel-id}/config/settings.json`

```json
{
  "id": "{channel-id}",
  "name": "{사용자가 정한 채널 소개 기반 채널명}"
}
```

#### 2. `channels/{channel-id}/config/workflow.json`

```json
{
  "mode": "ask"
}
```

→ 첫 채널은 기본 `ask`로 설정. 설명 추가:
```
모드를 "ask"로 설정했습니다.
- ask: 컨셉/훅/썸네일 단계에서 선택지를 보여주고 고르게 합니다
- auto: 전 과정 자동 (나중에 익숙해지면 바꿔보세요)
```

#### 3. `channels/{channel-id}/config/thumbnail-strategy.json`

라운드 4~8의 응답을 아래 형식으로 저장:

```json
{
  "count": 5,
  "art_styles": ["photorealistic", "anime"],
  "color_palette": "bright",
  "emotions": ["shock", "curiosity"],
  "text_space": "bottom-half",
  "face_style": "exaggerated",
  "brand": {
    "colors": [],
    "mascot": false,
    "fixed_layout": null
  },
  "benchmark_analysis": null
}
```

| 필드 | Q# | 값 매핑 |
|------|-----|---------|
| count | Q7 | A→3, B→5, C→7 |
| art_styles | Q8 | A→"photorealistic", B→"anime", C→커스텀 문자열 |
| color_palette | Q9 | A→"bright", B→"dark", C→"pastel", D→"neon", E→"monotone", F→"auto" |
| emotions | Q10 | A→"shock", B→"curiosity", C→"tension", D→"trust", E→"warmth", F→"humor", G→"premium" |
| text_space | Q11 | A→"bottom-half", B→"full", C→"left-right" |
| face_style | Q12 | A→"exaggerated", B→"natural", C→"silhouette", D→"minimize", E→"auto" |
| brand.colors | Q13-A | 사용자 입력 색상 배열 |
| brand.mascot | Q13-B | true/false |
| brand.fixed_layout | Q13-C | 사용자 입력 구도 설명 또는 null |
| benchmark_analysis | Q15 | 썸네일 분석 결과 요약 또는 null |

#### 4. `channels/{channel-id}/config/assets/` 디렉토리

```bash
mkdir -p channels/{channel-id}/config/assets/mascot
mkdir -p channels/{channel-id}/config/assets/style-reference
```

Q13에서 B(마스코트)를 선택했거나, Q14에서 레퍼런스가 있다고 한 경우에만 생성.

#### 5. `channels/{channel-id}/config/profile.md`

아래 포맷으로 생성. 파이프라인의 모든 에이전트가 이 파일을 참조한다.

```markdown
# {채널 소개 한 문장}

## 장르
{장르명}

## 톤 & 말투
- 스타일: {선택한 톤}
- 상세: {구체적 설명 — 예: "반말 베이스, 핵심 강조 시 존댓말 전환"}

## 금지 표현
- {표현 1}
- {표현 2}
- {표현 3}

## 시그니처 표현
- {표현 1} — {어떤 상황에서 사용}
- {표현 2} — {어떤 상황에서 사용}

## 서사 구조
- 선호: {서사 유형 1}, {서사 유형 2}
- 기피: {서사 유형} (이유: {간단한 이유})

## 관점
{항상 _____ 관점에서 이야기한다. 구체적 설명.}

## 정보 전달 방식
{선택한 방식 + 구체적 설명}

## 타겟 시청자
{장르 + 톤 + 관점에서 추론한 타겟 시청자 설명}
```

### 벤치마크 분석 결과 보존

벤치마킹 영상이 있었다면, `config/_benchmarks/` 폴더는 그대로 유지.
분석 요약을 `config/_benchmarks/analysis-summary.md`에 저장:

```markdown
# 벤치마크 분석 요약

## 분석 영상
1. {제목} — {조회수}, {채널명}
2. ...

## 공통 패턴
- 톤: ...
- 서사 구조: ...
- 훅 기법: ...
- 전달 방식: ...

## profile.md에 반영한 내용
- ...
```

## 필수 규칙

1. **폴더명/파일명에 한글 금지** — 영어 kebab-case만 사용
2. **추천을 먼저 제시** — 빈칸 주관식 질문 금지. 항상 선택지 또는 추천값 제공
3. **한 번에 3개 이상 질문 금지** — 라운드당 최대 2~3개
4. **벤치마킹은 선택사항** — 없어도 완전한 채널 생성 가능
5. **collect.py 실패 시** — "yt-dlp 업데이트 필요: pip install -U yt-dlp" 안내 후 벤치마킹 없이 진행
