# /verify — 프로젝트 전수 검증

`prompts/quality-gates.md` SSOT의 게이트 1~5 자동 실행. PD가 사용자에게 "완료" 보고하기 전 또는 사용자 직접 트리거.

## 사용법

```
/verify {프로젝트명}
```

예시:
- `/verify etf-ep07-part3`
- `/verify news-0427-evening`

## 실행 절차

### 1단계: 프로젝트 위치 자동 감지

`channels/*/projects/{프로젝트명}/` 경로 자동 매칭. 다중 채널이면 모두 검증.

### 2단계: 게이트 1 — 톤 전환 잔존 (3차 패턴 검색)

```
1차: 비판자|비판 댓글|한 분이|댓글창|그 댓글
2차: 비판 받|비판 들|비판 정면|비판 회수|비판자에|비판 핵심|비판 모드|비판 영상|비판 답변|비판 인정|비판 인용
3차: 비판\b (단독)
```

각 패턴별 잔존 건수 + 위치 보고.

### 3단계: 게이트 2 — YMYL 금융 정책 (게이트 4)

```
- "사라|팔아라|확정|100%|무조건|뚫는다|절대" 검색
- 종목 비중 권유 패턴 (예: "X 70/Y 30")
- 단정형 미래 보장
```

### 4단계: 게이트 3 — 출처 인라인 (게이트 4·9)

```
- 핵심 수치 카운트 (% / 만원 / $ / 원 등)
- 출처 인라인 명시 여부 (Total Real Returns, 한국경제, 기관 보고서 등)
- 출처 누락 비율 보고
```

### 5단계: 게이트 4 — 13게이트 🔴 5개 사전 점검

게이트 3·5·11·12·13 (허위/혐오/아동/자살/극단주의) 패턴 검색.
1건이라도 잔존 = 즉시 알림 + 사용자 confirm 강제.

### 6단계: 게이트 5 — 메타데이터 정합성

- 확정 제목 일관 (concept.md ↔ output/youtube.md ↔ output/thumbnails/prompts.json)
- 자기 발문 시작 톤 일관 (script.txt ↔ script-oneline.txt ↔ hook-intro.md ↔ youtube.md 설명)
- 핵심 수치 일관 (예: 76%p, 601%p가 모든 영상 노출 산출물에 존재)

### 7단계: 결과 매트릭스 표 출력

```
| 게이트 | 결과 | 잔존 위치 / 보강 제안 |
|---|---|---|
| 1. 톤 전환 잔존 | ✅ / ⚠️ / ❌ | ... |
| 2. YMYL 정책 | ✅ / ⚠️ / ❌ | ... |
| 3. 출처 인라인 | ✅ / ⚠️ / ❌ | ... |
| 4. 13게이트 🔴 | ✅ / ⚠️ / ❌ | ... |
| 5. 메타 정합성 | ✅ / ⚠️ / ❌ | ... |
```

### 8단계: 잔존 시 즉시 PD 직접 정리

발견된 잔존 표현은 PD가 직접 Edit으로 정리. 에이전트 재호출 X (다시 누락 가능성).

수정 완료 후 Grep 재검증 → 0건 확인 → 사용자 보고.

## 적용 우선순위

- 영상 제작 완료 직전 (사용자 "완료" 보고하기 전 PD 자율 검증)
- 사용자 직접 트리거 ("ep07-part3 검증해줘")
- 새 산출물 작성 후 즉시
- 톤 전환 / 일관성 보정 작업 종료 후
