# 대본 검수 체크리스트

## 핵심약속 검수

- [ ] Hook에서 핵심약속을 바로 이행하고 있는가 (감정 빌드업만 하고 미루지 않는가)
- [ ] 각 파트가 핵심약속 이행에 기여하는가 (핵심약속과 무관한 파트가 없는가)
- [ ] 마무리에서 핵심약속이 완결되는가

## 구조 검수

- [ ] outline.md의 본문 구조(파트 순서, 내용)대로 작성되었는가
- [ ] 확정 제목이 암시하는 구조를 정확히 따르는가 (예: "3가지 이유" → 정확히 3섹션)
- [ ] 타겟 러닝타임 범위 안에 들어가는가
- [ ] Hook과 Intro가 hook-intro.md와 일치하는가
- [ ] 각 파트의 감정 흐름이 outline.md 설계와 맞는가
- [ ] 리텐션 장치(오픈루프, 전환 멘트)가 파트 전환 지점에 배치되었는가
- [ ] verified-data.md의 검증된 데이터만 사용했는가

## 문체 검수

- [ ] 쉼표를 남발하지 않았는가 (이어붙일 수 있으면 이어붙인다)
- [ ] 따옴표를 남발하지 않았는가
- [ ] 초등학생도 이해할 수 있는 쉬운 말로 쓰였는가
- [ ] 용어 순화가 적용되었는가 (outline.md 용어 순화 매핑 참고)
- [ ] 한 문장에 숫자가 3개 이상 몰려있지 않은가 (비교용 2개까지는 OK)
- [ ] 분량이 목표 대비 130% 이상인 파트에 장황한 반복·불필요한 부연이 없는가 (같은 말 되풀이, 비유 연속 중첩, 이미 전달된 메시지 재설명 등)

## 톤 검수

- [ ] 채널 프로필(`prompts/channels/`)의 말투와 톤 규칙을 따랐는가
- [ ] outline.md에 정의된 프로젝트별 톤을 따랐는가

## 신규 주장 검수

- [ ] draft.md에 있지만 verified-data.md에 없는 사실적 주장(비교, 수치, 구체적 사례)이 있는가
- [ ] 있다면 각 주장을 WebSearch로 검증했는가
- [ ] 검증 결과를 아래 표에 빠짐없이 정리했는가

신규 주장이란: 검증 가능한 사실적 주장. 비교("A 돈이면 B를 할 수 있다"), 수치("X%가 Y이다"), 구체적 사례("Z 회사가 W했다") 등.
신규 주장이 아닌 것: 수사적 표현("취미가 아니라 짐이 됩니다"), 일반 상식("겨울이 짧아지고 있다"), 의견("어느 쪽을 고르겠습니까").
