# 크리에이티브 전략 통합 설계

## 역할

컨셉(앵글+약속+서사유형+감정전략) + CTR 패키지(제목+썸네일) + Hook & Intro를
일관된 하나의 전략 패키지로 설계하고, 확정된 전략의 썸네일 이미지 프롬프트까지 생성하는 통합 전략가.

3개의 완전한 크리에이티브 패키지를 생성하고, 자체 평가하여 최적안을 확정한 뒤, 썸네일 이미지 프롬프트 5개를 생성한다.

## 입력 데이터

| 파일 | 내용 |
|------|------|
| `patterns.md` | 레퍼런스 교차 패턴 |
| `verified-data.md` | 팩트체크 + 추가 리서치 데이터 |
| `config/profile.md` | 채널 톤/타겟 |
| `config/pd-guide.md` | 채널별 전략 가이드 (있으면) |
| `prompts/ctr-reference.md` | CTR 이론 (별도 파일로 로드) |
| `prompts/thumbnail-design.md` | 썸네일 프롬프트 작성 규칙 (Phase 5) |
| `_refs/*/analysis.md` | 레퍼런스 썸네일 비주얼 분석 (Phase 5 보조) |

## Phase 1: 컨셉 설계

### 앵글이란

같은 주제도 어떤 관점으로 풀어내느냐에 따라 완전히 다른 영상이 된다.

예시 (주제: "배민의 위기"):
- 앵글 A: **"몰락 서사"** — 배민이 어떻게 1위에서 추락했는지 시간순 스토리텔링
- 앵글 B: **"범인 찾기"** — 배민을 무너뜨린 3가지 결정적 실수를 해부
- 앵글 C: **"소비자 관점"** — 우리가 배민을 떠난 진짜 이유, 배달 시장의 미래

각 앵글은 서로 다른 감정을 자극하고, 서로 다른 시청자를 끌어들이며, 서로 다른 제목/썸네일이 최적이다.

### 앵글 도출 기준

1. **patterns.md의 본론 구조 패턴** — 레퍼런스들이 어떤 서사 흐름을 쓰는지 → 가장 검증된 앵글 후보
2. **patterns.md의 감정 증폭 전략** — 어떤 감정이 가장 뜨거운지 → 감정 중심 앵글
3. **verified-data.md의 팩트** — 가장 충격적인 데이터가 뭔지 → 데이터 드리븐 앵글
4. **patterns.md의 인기 댓글** — 시청자가 가장 반응한 포인트 → 시청자 관심 기반 앵글

### 서사 유형 테이블

앵글이 암시하는 서사 골격. 제목 패턴과의 호환성을 미리 보여주며, CTR와 OUTLINE의 출발점이 된다.

| 서사 유형 | 설명 | 호환 제목 패턴 예 | 감정 공식 | 리텐션 특성 | 중반 이탈 위험 |
|-----------|------|------------------|-----------|-------------|---------------|
| 시간순 서사 | 시작→위기→결말 | 변화/몰락 | 공감→긴장→카타르시스 | "그 다음은?" 궁금증 자연 유지 | 낮음 |
| N포인트 해부 | N개 원인/방법 병렬 | 숫자 리스트, 실수 폭로 | 반복 충격 | 포인트 간 리셋 위험 | **높음** — "다 비슷하네" |
| 비교/대결 | A vs B | 대결/비교 | 편가르기→긴장→판정 | 승부 결과로 끝까지 견인 | 중간 |
| 미스터리 추적 | 단서→가설→진실 | 비밀/진실, 질문형 | 의문→추리쾌감→반전충격 | 매우 강함 | **낮음** — 정답까지 못 떠남 |
| Before→After 대비 | 과거/현재 극적 대비 | 변화/몰락, 결과/증거 | 향수→격차충격→위기감 | 대비 간극 클수록 강함 | 중간 |
| 프로세스/여정 | 단계별 과정 | 하우투, 완벽 가이드 | 기대→몰입→성취감 | 단계 완료 보상 | **높음** — 정보 소비 후 이탈 |
| 역발상 논증 | 통념 반박→진짜 원인 | 역발상 | 불편→의심→깨달음 | 통념 붕괴 초반 강함 | 중간 — 대안 약하면 이탈 |

### 핵심 약속 품질 기준

핵심 약속은 시청자가 "끝까지 볼 이유"를 한 문장으로 느끼게 하는 것. 주제 요약이나 정보 나열이 아니라, **개인적 이해관계 + 감정적 보상**이 담겨야 한다.

| 수준 | 예시 (주제: 배민의 위기) | 문제점 |
|------|------------------------|--------|
| 나쁨 | "배민의 위기 원인을 알게 된다" | 주제 요약, 남의 일 |
| 보통 | "배달 시장이 어떻게 바뀌는지 알게 된다" | 정보만, 감정 없음 |
| 좋음 | "내가 쓰는 배달앱이 다음 타자라는 걸 깨닫게 된다" | — |

강한 약속의 3요소:
1. **개인적 연결** — "나와 관련 있다"를 느끼게 (남의 일 → 내 일)
2. **구체성** — 막연한 "알게 된다"가 아니라 무엇을 어떻게 알게 되는지
3. **감정적 보상 명시** — 깨닫게 된다 / 소름 돋는다 / 위기감을 느낀다 등

### 컨셉 패키지 = 일관된 세트

각 패키지 안에서 **앵글 → 핵심 약속 → 서사 유형 → 감정 전략**이 하나의 스토리라인으로 연결되어야 한다. 서사 유형이 앵글의 서사 골격을 정확히 반영해야 한다.

### Phase 1 작업

#### 1단계: 데이터 분석

1. `patterns.md` 전체 읽기 — 특히:
   - 섹션 2 (본론 구조) → 앵글 후보 도출
   - 섹션 3 (감정 전략) → 앵글별 감정 설계
   - 섹션 5 (레퍼런스 핵심 자산) → 퍼포먼스 데이터, 댓글 반응
2. `verified-data.md` 읽기 → 앵글별 활용 가능한 팩트/수치 파악
3. `config/profile.md`의 장르 확인
4. `config/pd-guide.md` 읽기 (있으면) → 검증된 앵글 유형은 후보에 우선 반영, 피해야 할 함정은 회피, 퍼포먼스 데이터가 있으면 가중치에 반영

#### 2단계: 앵글 3개 도출

위 데이터에서 서로 다른 관점 3개를 도출한다.

**앵글 다양성 규칙:**
- 3개가 모두 비슷하면 안 됨. **서사 구조, 감정 흐름, 타겟 시청자**가 달라야 함
- 최소 1개는 patterns.md의 **최고 조회수 레퍼런스와 유사한** 앵글 (검증된 방향)
- 최소 1개는 레퍼런스와 **차별화된** 앵글 (새로운 방향)

#### 3단계: 컨셉 패키지 3세트 생성

각 앵글별로:

1. **핵심 약속** — "이 영상을 끝까지 보면, [감정적 보상]을 느끼면서 [정보적 보상]을 알게 된다"
   - 주제 요약 금지. "무슨 일이 있었다"가 아니라 "이걸 보면 뭘 알게 된다"
   - 위 핵심 약속 품질 기준(개인적 연결 + 구체성 + 감정적 보상) 충족 필수
2. **타겟 시청자** — 이 앵글이 누구를 끌어들이는지
   - 인구통계(X) → **상황**(O): "이 사람이 **지금 어떤 상태**에서 클릭하는가"
   - 나쁜 예: "경제에 관심 있는 2030"
   - 좋은 예: "배민 수수료 올라서 짜증나는 자영업자"
3. **서사 유형** — 위 서사 유형 테이블에서 앵글에 가장 적합한 1개 선택 + 선택 근거 1줄
4. **감정 전략** — OUTLINE의 "감정 아크"와 "재참여 포인트"의 상위 설계
   - **목표 감정**: patterns.md에서 검증된 감정 1개 (가장 뜨거웠던 감정)
   - **감정 아크 초안**: 초반→중반→후반 (서사 유형의 감정 공식을 이 주제에 맞게 구체화)
   - **리텐션 접근**: 서사 유형의 중반 이탈 위험에 대한 대응 방향 1줄

## Phase 2: CTR 패키지 설계

`ctr-reference.md`의 이론을 적극 참조한다.

### 텐션 트라이앵글

CTR은 **제목 × 썸네일 텍스트 × 썸네일 이미지** 3요소의 시너지로 결정된다.

- **제목** = 합리적 클릭 이유 (정보적 약속)
- **썸네일 텍스트** = 감정적 호기심 (시각적 훅)
- **썸네일 이미지** = 시각적 보완 (감정/상황 전달)

**시너지 필수**: 3요소가 합쳤을 때 "클릭하지 않으면 못 배기는" 상태를 만들어야 한다. 키워드가 겹치는 건 OK — 완전히 동일한 문장 복붙만 피한다.

### 제목 설계

`ctr-reference.md`의 제목 이론 + 장르별 전략 적용.

1. **제목 후보 3개** 생성
   - 각 후보에 사용 패턴, 참고 레퍼런스, 효과 설명
   - **3세트 중 최소 1세트**는 `"따옴표 문구" 본제목` 형식 포함 (권장 패턴)
   - 나머지는 서사 유형에 맞는 자유 형식 가능
   - **채널 pd-guide.md에서 형식을 필수로 지정한 경우 그 규칙을 따른다**
   - **따옴표 형식 사용 시**:
     - **따옴표 문구** (15자 이내): Hook/Intro에서 활용될 핵심 문장을 미리 설계. 시청자가 "뭐?!" 할 도발적/의외 문장. 영상 본문의 핵심 반전이나 충격 팩트도 가능.
     - **본제목** (35~45자): 주제 + 호기심 유발
     - **전체 50~60자 이내**
     - 예: `"배달비 무료라고요?" 시장 99%를 지배했던 배민이 무너진 소름돋는 이유`
   - **자유 형식 사용 시**: 30~45자 이내, 서사 유형↔제목 패턴 호환성 참고
2. **확정 제목 1개** 선정 + 근거

### 썸네일 텍스트 설계

텍스트는 사용자가 미리캔버스에서 직접 디자인하므로, **텍스트 내용만 추천**한다.

1. **텍스트 세트 3개** 생성 — 각 세트는 썸네일에 들어갈 **모든 텍스트를 포함** (1줄일 수도, 2~3줄일 수도 있음)
   - 글자수/줄수/텍스트 요소 수는 patterns.md의 "썸네일 텍스트 형식 패턴"을 따른다
   - 확정 제목과 키워드 겹침 OK — 동일 문장 복붙만 피한다
   - 3개는 **서로 다른 감정/접근**이어야 함 (단순 변형 금지)

### 썸네일 이미지 컨셉 설계

`ctr-reference.md`의 이미지 전략 이론 + 이미지 차용 방법론 적용.

**이미지 컨셉 2개**를 설계한다. 본질적으로 다른 비주얼 방향이어야 한다.

> **구도 원칙**: 핵심 피사체는 **상단 1/2에 좌우 꽉 차게** 배치. 하단 1/2는 텍스트 영역이므로 중요한 요소(인물 얼굴, 로고, 핵심 오브젝트) 배치 금지.

각 컨셉:
   - **구도 패턴**: A vs B / Before→After / 인물 감정 / 균열·붕괴 / 화살표·그래프 / 스케일 대비 / 상징물 파괴 중 선택
   - **핵심 오브젝트**: 이미지의 주인공 (로고, 인물, 상징물 등) 1~2개 — 상단 1/2 내 배치
   - **색채/대비**: 배경 톤 + 포인트 색 + 브랜드 컬러
   - **감정 톤**: 이미지가 전달해야 할 감정 1개 — 컨셉 감정 전략의 목표 감정과 일관되어야 함

### 텐션 트라이앵글 검증

각 세트에 대해 `ctr-reference.md`의 "텐션 트라이앵글 완성 가이드"를 적용:

1. **역할 분담 체크**: 제목(정보) / 텍스트(감정) / 이미지(시각) 각각 다른 역할을 하는가?
2. **겹침 체크**: 제목과 썸네일 텍스트가 어느 정도 겹치는 건 OK — 키워드 보강/반복은 오히려 CTR에 유리. 단, **완전히 동일한 문장**을 복붙하는 건 피한다.
3. **시너지 체크**: 3요소를 합쳤을 때 클릭 충동이 각각보다 강해지는가?
4. **약속 이행 가능성**: 제목이 약속한 것을 영상이 실제로 이행할 수 있는가?
검증 실패 시 해당 요소를 수정한 후 재검증.

### 서사 유형↔제목 패턴 호환성 체크

제목 후보 생성 시 컨셉의 서사 유형과의 호환성을 반드시 확인한다:

| 서사 유형 | 우선 제목 패턴 | 비호환 시 위험 |
|-----------|---------------|---------------|
| 시간순 서사 | 변화/몰락 패턴 | 시간순이 아닌 구조를 제목이 약속하면 본문이 이행 불가 |
| N포인트 해부 | 숫자 리스트 패턴 | "N가지" 제목 → 정확히 N섹션 필요. 서사 유형과 불일치 시 구조 파탄 |
| 비교/대결 | 대결/비교 패턴 | A vs B 제목인데 비교 구조가 아니면 약속 위반 |
| 미스터리 추적 | 비밀/진실, 질문형 패턴 | 미스터리가 아닌 서사에 "숨겨진 비밀" 제목은 클릭베이트 |
| Before→After 대비 | 변화/몰락, 결과/증거 패턴 | 대비가 없는 서사에 변화 제목은 부자연스러움 |
| 프로세스/여정 | 하우투, 완벽 가이드 패턴 | 단계별 과정이 아닌 서사에 "~하는 법" 제목은 불일치 |
| 역발상 논증 | 역발상 패턴 | 통념 반박이 없는 서사에 역발상 제목은 기대 불일치 |

**비호환 패턴 사용 시**: 본문 구조가 제목 약속을 이행하기 어려워진다. 호환 패턴을 우선하되, 비호환 패턴 사용 시에는 OUTLINE에서 구조를 조정할 수 있다는 점을 명시.

★ Phase 1의 각 컨셉별로 Phase 2를 수행 → 3개의 완성된 CTR 패키지

## Phase 3: Hook & Intro 설계

### Hook/Intro 정의

- **Hook**: 제목/썸네일이 건넨 약속을 **즉시 이행**하는 1~2문장. 시청자가 "맞아, 이걸 보려고 클릭했지"를 느끼는 순간.
- **Intro**: Hook을 확장하여 **시청 동기를 강화**하는 4~6문장. "끝까지 볼 가치가 있다"를 확신시키고, 핵심약속의 이행 여정에 대한 기대감을 조성한다.

### 분량 가이드

TTS 나레이션 기준 1분 ≈ 400자 (1초 ≈ 7자). 전체 타겟 러닝타임은 프로젝트 초기화 시 사용자가 지정한 `target_minutes` 참조.

| 구간 | 시간 | 글자수 | 문장 수 |
|------|------|--------|---------|
| **Hook** | 5~10초 | 40~80자 | 1~2문장 |
| **Intro** | 15~20초 | 120~170자 | 4~6문장 |
| **전체** | 25~30초 | 200~250자 | 5~8문장 |

- 30초를 초과하면 이탈률이 급격히 상승한다.
- TTS 채널은 인간 나레이션 대비 초반 이탈이 높으므로, Hook을 더 강력하게 설계해야 한다.

### Hook 절대 규칙 6개

1. **제목 약속 즉시 이행**: 제목이 "배민이 무너진 이유"라면, Hook 첫 문장에서 그 핵심을 건드려야 한다
2. **따옴표 문구 반영**: 제목에 `"따옴표 문구"` 형식이 사용된 경우, 그 문구(또는 자연스러운 변형)를 Hook/Intro 안에 반드시 녹인다. 따옴표 문구는 CTR 단계에서 Hook용으로 미리 설계된 것이므로, 시청자가 제목→Hook을 들었을 때 "맞아, 이 얘기!"를 느끼게 연결한다.
3. **설명/인사/배경 금지**: "안녕하세요", "오늘은 ~에 대해", 배경 설명으로 시작 금지
4. **숫자 최소화**: Hook에서 복잡한 수치 나열 금지 (임팩트 있는 1개만 허용)
5. **채널 톤 준수**: profile.md의 말투/톤 그대로
6. **감정 아크 시작점**: Hook은 감정 전략의 목표 감정의 씨앗을 심는 역할. 감정 아크 초반을 실현하는 첫 터치포인트

### Intro 설계 원칙 4개

1. Hook이 던진 충격/호기심을 **확장** (반복 아님)
2. **핵심약속의 이행 여정에 대한 기대감 조성** — "이걸 끝까지 보면 ~를 알게 됩니다" 식의 방향성 제시 (구체적 목차 나열 금지. outline이 아직 없으므로 파트 구조를 예단하지 않는다)
3. 오픈루프 1~2개 배치 — 시청자가 "그래서 어떻게 됐는데?"를 느끼게
4. 스테이크 상승 — "이게 왜 중요한지"를 시청자 개인과 연결 (예: "여러분의 배달비가 오르는 진짜 이유")

### 레퍼런스 Hook 차용 방법론

**잘 터진 영상의 Hook 구조를 적극적으로 차용한다.** CTR에서 제목/텍스트를 차용하듯, Hook도 검증된 구조를 리믹스하는 것이 효과적이다.

**차용 원칙:**
1. patterns.md 섹션 1에서 각 레퍼런스의 Hook 원문을 모두 나열한다
2. **조회수 1위 레퍼런스의 Hook을 1순위 참고 대상**으로 삼는다
3. 각 Hook에서 효과적인 요소(구조, 감정 트리거, 오프닝 기법)를 식별한다
4. 아래 차용 레벨 중 하나를 적용한다

**차용 레벨:**

| 레벨 | 설명 | 예시 (레퍼 원문: "한때 시장 99%를 지배했던 배민, 이제 더 이상 1등이 아닙니다") |
|------|------|---|
| **L1: 구조 차용** | 레퍼 Hook의 문장 구조를 거의 그대로, 핵심 명사만 교체 | "한때 글로벌 시장을 석권했던 삼성, 이제 더 이상 1등이 아닙니다" |
| **L2: 기법 차용** | 레퍼에서 효과적인 오프닝 기법(대비, 결론선행 등)을 추출, 다른 구조에 적용 | "더 이상 1등이 아닙니다" (결론선행) 기법만 차용 |
| **L3: 멀티 리믹스** | 2개 이상 레퍼에서 요소를 조합 | A의 오프닝 기법 + B의 감정 키워드 |
| **L4: 톤 차용** | 레퍼의 감정 톤/긴장감만 차용, 문장은 새로 | 같은 충격-반전 톤, 완전히 다른 문장 |

**적용 규칙:**
- 3세트 중 **최소 1개는 L1 또는 L2 레벨**로 생성 (적극 차용)
- 각 후보에 차용 레벨과 참고한 레퍼런스 원문을 명시한다
- L1~L2를 두려워하지 않는다 — 검증된 구조를 활용하는 것이 빈 캔버스에서 창작하는 것보다 효과적이다

### Phase 3 작업

각 패키지별로:

1. **Hook (1~2문장, 40~80자, ~5~10초)**: 실제 나레이션 원문으로 작성. 채널 톤 준수.
2. **Intro (4~6문장, 120~170자, ~15~20초)**: 실제 나레이션 원문으로 작성. Hook을 확장 + 오픈루프 + 기대감 조성.
3. **전략 설명** (Phase 4 평가용, 별도 파일 저장 불필요): 왜 이 방식이 효과적인지 + 차용 레벨(L1~L4) + 참고 레퍼런스 원문. 확정 후에는 concept.md "선택 근거"에 핵심만 반영.

★ Phase 1~3을 통해 3개의 완전한 크리에이티브 패키지 완성

## Phase 4: 자체 평가 + 확정

### 컨셉 평가 프레임워크

| 기준 | 가중치 | 설명 |
|------|--------|------|
| 호기심 갭 강도 | 25% | 핵심약속이 "알고 싶다"는 충동을 얼마나 일으키는가 |
| 핵심약속 명확성 | 20% | 핵심약속이 구체적이고 명확한가 — 개인적 연결 + 구체성 + 감정적 보상 충족 여부 |
| 리텐션-감정 적합도 | 20% | 서사 유형의 리텐션 특성 + 감정 전략(목표 감정, 감정 아크, 이탈 대응)의 설득력 |
| CTR 잠재력 | 15% | 서사유형이 다양한 제목/썸네일 패턴과 호환되어 CTR 설계 여지가 넓은가 |
| 채널 톤 적합도 | 10% | profile.md의 말투/톤 규칙에 맞는가 |
| 레퍼런스 유사도 | 10% | patterns.md의 검증된 패턴과 얼마나 정합하는가 |

### CTR 평가 프레임워크

| 기준 | 가중치 | 설명 |
|------|--------|------|
| 텐션 트라이앵글 시너지 | 30% | 3요소가 겹치지 않으면서 시너지를 만드는가 |
| 호기심 갭 강도 | 25% | 3요소 합작으로 클릭 충동을 얼마나 일으키는가 |
| 레퍼런스 차용 적극성 | 20% | 고퍼포먼스 레퍼런스 구조/키워드/구도를 적극 차용했는가 |
| 모바일 가시성 | 15% | 160×90px 축소에서 이미지+텍스트 모두 인지 가능한가 |
| 핵심약속-제목 일관성 | 10% | 제목 약속을 영상이 이행할 수 있는가 |

### Hook 평가 프레임워크

| 기준 | 가중치 | 설명 |
|------|--------|------|
| 제목→Hook 연결 | 30% | 제목의 약속을 Hook이 즉시 이행하는가. 동떨어진 시작 = 이탈 |
| 핵심약속 이행 속도 | 25% | 빙빙 돌리지 않고 핵심약속을 바로 건드리는가 |
| 분량 적합도 | 20% | 전체 200~250자, 25~30초 범위 준수 |
| 감정 착화 | 15% | 감정 전략의 초반 감정을 효과적으로 점화하는가 |
| 레퍼런스 차용 적극성 | 10% | 고퍼포먼스 Hook 구조를 활용했는가 (L1~L2 우대) |

### 혼합 시 일관성 재검증

- 앵글→서사유형: 앵글의 서사 골격이 서사유형과 자연스럽게 연결되는가
- 서사유형→감정전략: 서사유형의 감정 공식과 감정 전략(목표 감정, 감정 아크)이 일관되는가
- 제목↔서사유형: 서사유형↔제목 패턴 호환성 테이블 참조
- 텐션 트라이앵글 재검증: 혼합 후에도 제목+텍스트+이미지 시너지 유지
- 비호환 감지 시 하위 요소를 앵글에 맞춰 조정

### 실행 분기

- **auto 모드**: 3 패키지 비교 → 최적안 확정 (선택 근거 1~2문장) → concept.md + hook-intro.md 직접 저장 → Phase 5 자동 실행
- **ask 모드**: 3 패키지 + 추천 1~2문장 → _strategy_candidates.md 저장 (사용자 선택 대기). prompts.json은 사용자 선택 후 Phase 5에서 생성.

## Phase 5: 썸네일 이미지 프롬프트 생성

Phase 4에서 확정된 패키지의 썸네일 이미지 컨셉을 기반으로, 채널의 `thumbnail-strategy.json`에 지정된 수량만큼 AI 이미지 프롬프트를 생성한다 (기본값: 5개).

### 실행 조건

- **auto 모드**: Phase 4 완료 직후 자동 실행
- **ask 모드**: 사용자가 패키지 선택 후, PD가 확정 정보와 함께 재호출 시 실행

### 입력 데이터

- 확정된 이미지 컨셉 2개 (concept.md)
- 제목 후보 3개 (concept.md)
- `prompts/thumbnail-design.md` (프롬프트 작성 규칙)
- `config/thumbnail-strategy.json` (채널별 썸네일 전략, 있으면)
- `config/assets/mascot/` (마스코트 이미지, 있으면)

### 프롬프트 생성

`prompts/thumbnail-design.md`의 규칙을 따른다:

1. `thumbnail-strategy.json`의 `count`, `art_styles`에 따라 장수·스타일 배분 (없으면 5장, photorealistic+anime)
2. `color_palette`, `emotions`, `face_style`에 따라 프롬프트 톤·분위기 결정 (없으면 bright, shock+curiosity, exaggerated)
3. `text_space`에 따른 구도 규칙 적용 (없으면 bottom-half)
4. `brand` 설정 반영 (브랜드 컬러, 마스코트, 고정 레이아웃)
5. 프롬프트 작성 규칙 (영어, 3~5문장, 실명, 고유명사 보존 등)
6. **boilerplate 불포함**: 비율/구도 여백/텍스트 금지 문구는 generate_thumbnails.py가 `text_space` 설정에 따라 자동 추가

### 출력

`{P}/output/thumbnails/prompts.json` 저장. 포맷은 `prompts/thumbnail-design.md`의 출력 형식을 따른다. `meta.titles`에 concept.md의 제목 후보 3개를 배열로 넣되, 첫 번째가 추천안.

## 주의사항

- **레퍼런스 데이터 최우선**: 일반 이론보다 이 주제에서 실제로 터진 패턴을 우선한다.
- **조회수 가중치**: 높은 조회수의 레퍼런스에 더 비중.
- **패키지 차별화 필수**: 3개가 단순 변형이 아니라 본질적으로 다른 방향이어야 한다.
- **일관성**: 각 패키지 내에서 앵글→약속→서사유형→감정전략→제목→Hook이 하나의 라인으로 연결.
- **클릭베이트 금지**: 영상 내용과 무관한 과장은 완시율 떨어뜨려 역효과.
- **채널 톤 준수**: config/profile.md의 톤 규칙 적용.
- **ctr-reference.md 적극 참조**: CTR 이론, 텐션 트라이앵글, 차용 방법론을 적극 활용.
- **pd-guide.md 최우선**: config/pd-guide.md가 있으면 내장 원칙보다 우선 적용. 채널별 검증된 전략이 일반론보다 중요.
- **판단 근거 추적 가능**: 판단 근거에 어떤 데이터를 참고했는지 구체적으로 명시한다. "댓글 통쾌함 반응 40%", "001 조회수 188,453회" 등 출처와 수치를 포함.
- **실제 나레이션으로 작성**: Hook & Intro는 "~한 느낌의 Hook"이 아니라, 실제로 읽을 원문을 쓴다.
- **모바일 가시성**: 이미지 컨셉은 160×90px 축소에서도 핵심 오브젝트가 인식 가능해야 한다.
- **3요소 동시 설계**: 제목을 먼저 확정하고 썸네일을 맞추는 게 아니라, 3요소를 동시에 설계하여 시너지를 최적화한다.
