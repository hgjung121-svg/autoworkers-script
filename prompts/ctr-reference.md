## 레퍼런스 전수 조사 가이드

patterns.md의 "반복되는 썸네일 & 제목 패턴" 섹션에서 추출:

1. **각 레퍼런스의 썸네일 텍스트 원문** — 모든 텍스트 요소를 **글자 그대로** 나열 (patterns.md의 썸네일 텍스트 원문 비교표 참조)
2. **각 레퍼런스의 제목** — 구조 분해 (패턴 식별)
3. **조회수 상관관계** — 어떤 텍스트/표현이 고퍼포먼스와 연결되는지
4. **제목-썸네일 역할 분담** — 각 레퍼런스에서 제목과 썸네일 텍스트가 어떻게 나눠 먹는지

⚠️ **레퍼런스 텍스트 원문이 이후 리믹스의 핵심 재료다.** 정확히 옮기지 않으면 리믹스 품질이 떨어진다.

## 이론: 제목 공식

### 3대 원리

**호기심 갭**: 시청자가 아는 것과 알고 싶은 것 사이의 긴장. 정보를 암시하되 완전히 공개하지 않아 클릭 충동 유발.

**텐션 트라이앵글**: 썸네일(감정적 호기심) + 제목(합리적 클릭 이유) + 훅(약속 이행). 셋이 시너지를 만들어야 한다. 제목과 썸네일이 같은 말을 하면 정보 낭비.

**IMPACT**: Immediate Hook(처음 3-5단어) + Measurable Outcome(숫자/결과) + Curiosity/Controversy(궁금증). 2-3개 조합이면 충분.

### 12가지 검증된 제목 패턴

| # | 패턴 | 템플릿 | 적합 장르 |
|---|------|--------|----------|
| 1 | 숫자 리스트 | [숫자] + [주제] + [한정/이점] | 교육, 정보 |
| 2 | 하우투 | ~하는 법 + [한정어] | 교육, 테크 |
| 3 | 대결/비교 | [A] vs [B] + [맥락] | 리뷰, 경제 |
| 4 | 변화/몰락 | [시작] → [결과] + [방법/기간] | 경제, 스토리 |
| 5 | 실수 폭로 | [대상]이 저지른 실수 | 경제, 교육 |
| 6 | 역발상 | 왜 [통념]이 틀렸는가 | 경제, 시사 |
| 7 | 비밀/진실 | [대상]이 말하지 않는 [진실] | 시사, 경제 |
| 8 | 도전/실험 | [행동]을 [기간] 해봤더니 | 엔터, 실험 |
| 9 | 완벽 가이드 | [주제] 완벽 정리 | 교육, 정보 |
| 10 | 질문형 | [도발적 질문] | 시사, 토론 |
| 11 | 결과/증거 | [구체적 결과] + [방법] | 리뷰, 교육 |
| 12 | 스토리 훅 | [흥미 설정] + [결과/위험] | 야담, 스토리, 경제 |

### 데이터 기반 규칙

- **모바일 잘림 기준**: 전체 60자 이내. 따옴표 형식 시 50~60자, 미사용 시 30~45자
- **숫자 사용**: 장편 영상 CTR 32-40% 향상 (홀수 > 짝수)
- **확정형 > 의문형**: 확정형 서술("추락하는 배민")이 의문형("어쩌다?")보다 클릭 유도 강함
- **괄호/따옴표**: 1회 사용 시 CTR 소폭 향상 ("1위가 바뀌었다" 같은 인용 느낌)
- **물음표**: 약한 성과 (호기심보다 불확실성으로 인식)

## 이론: 썸네일 문구 공식

### 핵심: 제목과의 관계

- 썸네일 텍스트 = **감정적 호기심** (시각적 훅)
- 제목 = **합리적 클릭 이유** (정보적 약속)
- 키워드가 겹치는 건 OK (오히려 메시지 보강) — 단, **완전히 동일한 문장을 복붙하지 않는다**
- 겹침을 피하려고 썸네일 텍스트를 억지로 줄이거나 빈약하게 만들지 않는다

### 문구 규칙

1. **글자수/줄 수는 레퍼런스를 따른다**: patterns.md의 "썸네일 텍스트 형식 패턴"에서 도출된 평균 텍스트 요소 수, 요소당 글자수, 총 글자수, 줄 수를 기준으로 삼는다. 고정 글자수 규칙 없음 — **레퍼런스가 3줄이면 3줄, 레퍼런스가 요소당 8~10자면 8~10자**.
2. **요약이 아니라 훅**: 내용 설명 X, 감정 반응 유발 O
3. **오픈 루프**: 답을 주면 스크롤. 문구가 루프를 열어야 한다
4. **감정 키워드 우선**: "몰락", "진실", "충격", "반전"
5. **텍스트 구조는 레퍼런스를 따른다**: 레퍼런스가 핵심+보조 2개 구조면 2개, 3개 독립 텍스트면 3개. 레퍼런스의 배치 방식(단일 블록/분산/계층형)도 참고한다.

### 문구 패턴

| 패턴 | 예시 | 효과 |
|------|------|------|
| 결과 선언형 | "몰락", "끝났다", "역전" | 강한 감정 즉시 유발 |
| 경고형 | "절대 하지 마", "속고 있다" | 공포/불안 |
| 비밀 폭로형 | "진짜 이유", "숨겨진 진실" | 호기심 갭 극대화 |
| 수치 충격형 | "7000억", "90% 독점" | 구체성 + 충격 |
| 대비형 | "A vs B", "전 vs 후" | 시각적 드라마 |

## 장르별 전략

config/profile.md의 `## 장르`를 읽고 아래에서 매칭. 정확히 일치하지 않으면 가장 가까운 것 적용.

### 경제/시사
- **제목**: 결과 스포일 + 원인 암시 ("진짜 이유", "이유가 있다")
- **썸네일**: 위기/몰락 키워드 + 브랜드/인물
- **감정**: 분노 → 통쾌 → 경각심
- **우선 패턴**: #4(변화/몰락) > #7(비밀/진실) > #6(역발상) > #12(스토리훅)

### 야담/역사/스토리
- **제목**: 인물 중심 + 극적 결말 암시 ("~의 최후", "~가 벌어졌다")
- **썸네일**: 인물/상황 + 긴장감 키워드
- **감정**: 몰입 → 긴장 → 카타르시스
- **우선 패턴**: #12(스토리훅) > #4(변화/몰락) > #7(비밀/숨겨진)

### 교육/정보
- **제목**: 실용적 이익 명시 ("~하는 법", "~만 알면")
- **썸네일**: 결과/비포에프터 + 숫자
- **감정**: 기대감 → 자신감
- **우선 패턴**: #2(하우투) > #1(숫자리스트) > #5(실수폭로)

### 엔터테인먼트/리뷰
- **제목**: 감탄/충격 + 구체적 대상
- **썸네일**: 강한 표정 + 짧은 반응어
- **감정**: 호기심 → 놀라움
- **우선 패턴**: #8(도전) > #11(결과) > #3(대결)

### 테크/IT
- **제목**: 제품/서비스명 + 평가/비교
- **썸네일**: 제품 이미지 + 평가 키워드
- **감정**: 궁금증 → 확신
- **우선 패턴**: #3(대결) > #2(하우투) > #9(완벽가이드)

## 제목 생성 규칙

1. **레퍼런스 패턴 최우선**: patterns.md에서 추출된 "이 주제에서 통하는" 제목 구조를 먼저 적용
2. **장르 패턴 적용**: 채널 장르의 우선순위 패턴 활용
3. **호기심 갭 필수**: 모든 후보에 "알고 싶게 만드는" 요소
4. **확정형 서술 우선**: 의문형보다 확정형
5. **핵심 키워드 포함**: 검색/추천 최적화

각 후보마다:
- 사용한 패턴 명시
- 참고한 레퍼런스
- 효과 한 줄 설명

### 선정 기준

1. 레퍼런스 최고 조회수 영상의 제목 구조와 가장 유사
2. 호기심 갭이 가장 강함
3. 핵심약속과 일치
4. 모바일 잘림 기준 60자 이내 (따옴표 형식 50~60자, 미사용 30~45자)

## 썸네일 문구 차용 방법론

### ★ 레퍼런스 적극 차용 ★

**잘 터진 영상의 썸네일 문구를 적극적으로 차용한다. 핵심은 "검증된 문구를 우리 주제에 맞게 변형"하는 것이다.**

**차용 원칙**:

1. 레퍼런스 썸네일 텍스트 원문을 모두 나열한다
2. **조회수 1위 레퍼런스의 문구를 1순위 참고 대상**으로 삼는다
3. 각 텍스트에서 효과적인 요소(키워드, 구조, 감정)를 식별한다
4. 아래 차용 레벨 중 하나를 적용한다

**차용 레벨** (위가 가장 적극적):

| 레벨 | 설명 | 예시 (레퍼 원문: "쿠팡에게 1위를 뺏긴") |
|------|------|----------------------------------------|
| **L1: 구조 차용** | 레퍼 문구의 구조를 거의 그대로 가져오고 핵심 명사만 교체 | "테무에게 시장을 뺏긴" |
| **L2: 키워드 차용** | 레퍼에서 효과적인 키워드를 뽑아 다른 구조에 배치 | "뺏긴 왕좌" |
| **L3: 멀티 리믹스** | 2개 이상 레퍼에서 요소를 조합 | A키워드 + B구조 |
| **L4: 감정 차용** | 레퍼의 감정 톤만 가져오고 표현은 새로 | 같은 위기감, 다른 표현 |

**핵심: L1~L2를 두려워하지 않는다.** 잘 터진 문구의 구조와 키워드를 적극 차용하는 것이 CTR에 유리하다. 완전히 동일한 문장만 아니면 된다.

**리믹스 패턴** (L3 적용 시):
- **A의 키워드 + B의 구조**: "몰락"(A) + "[대상]에게 1위를 뺏긴"(B) → "쿠팡에게 1위를 뺏긴 배민의 몰락"
- **A의 감정 + B의 정보**: "추락하는"(A) + "배달 시장"(B) → "추락하는 배달 제국"
- **A+B+C 하이브리드**: "무너지는"(A) + "배민"(공통) + "진짜 이유"(C) → "무너지는 배민, 진짜 이유"

### 생성 규칙

⚠️ **글자수, 텍스트 요소 수, 줄 수는 모두 patterns.md의 "썸네일 텍스트 형식 패턴"을 따른다.**
- 레퍼런스 평균이 요소 3개 × 요소당 7~10자면 → 후보도 요소 3개 × 7~10자로 생성
- 레퍼런스 평균이 핵심+보조 2개 × 4~6자면 → 후보도 2개 × 4~6자로 생성

각 후보마다:
- 텍스트 요소별 글자수 명시 (레퍼런스 패턴과 비교)
- 참고한 레퍼런스 원문 인용
- 차용 레벨 (L1~L4) 명시
- 원문에서 무엇을 가져왔는지 구체적으로 설명
- **최소 1개는 L1 또는 L2 레벨**로 생성 (적극 차용)

### 선정 기준

1. 확정 제목과 **보완 관계** (키워드 겹침 OK, 동일 문장 복붙만 피한다)
2. **최고 조회수 레퍼런스의 문구와 가장 유사** (L1~L2 우선 검토)
3. **레퍼런스 형식 패턴과 일치**: 텍스트 요소 수, 요소당 글자수, 총 글자수가 레퍼런스 평균에 근접
4. 감정 반응이 가장 강함

## 이론: 썸네일 이미지 전략

### 검증된 구도 패턴

| 패턴 | 설명 | 적합 장르 | 예시 |
|------|------|-----------|------|
| **A vs B 대결** | 좌우 분할, 두 대상의 대비 | 경제, 비교, 시사 | 배민 로고 vs 쿠팡이츠 로고, 밝은면 vs 어두운면 |
| **Before → After** | 시간/상태 변화를 좌→우 또는 상→하로 | 경제, 스토리 | 전성기(밝은 톤) → 몰락(어두운 톤) |
| **인물 감정** | 인물이 카메라를 정면 응시 + 강한 표정 | 전 장르 | 충격받은 얼굴, 걱정하는 표정, 분노 |
| **균열/붕괴** | 건물, 로고, 상징물에 균열이 가는 이미지 | 경제, 시사 | 기업 건물에 균열, 부서지는 왕관 |
| **화살표/그래프** | 하락↓, 상승↑, 역전 교차를 시각화 | 경제, 데이터 | 빨간 하락 화살표, 교차하는 두 선 |
| **스케일 대비** | 크기 차이로 힘의 관계 표현 | 경제, 시사 | 거대한 A가 작은 B를 내려다봄 |
| **상징물 파괴** | 핵심 브랜드/상징의 파괴 이미지 | 경제, 스토리 | 부서지는 배달 박스, 갈라지는 앱 아이콘 |

### 색채 규칙

- **배경**: 어두운 톤 (검정, 남색, 짙은 회색) — 드라마틱한 분위기
- **포인트 색**: 빨강(위기/긴급), 노랑(주의/강조), 초록(성장/돈)
- **브랜드 컬러 활용**: 주제에 등장하는 기업/브랜드의 시그니처 컬러 적극 사용
- **고대비 필수**: 밝은 오브젝트 + 어두운 배경, 또는 따뜻한 색 + 차가운 색 대비
- **채도 높게**: 유튜브 피드에서 눈에 띄려면 채도가 높아야 함

### 한국 시장 특화

- **한국 브랜드 컬러**: 배민=민트, 쿠팡=로켓파란색, 카카오=노랑, 삼성=파랑, 네이버=초록 등
- **한국적 맥락**: 한국인이 즉시 인식하는 심볼/장소/오브젝트 사용
- **인물 표현**: 실존 인물은 실명+직함을 영문으로 표기 (예: `Lee Jae-yong, Samsung Electronics chairman`). 일반화 금지.
- **배달/음식 문화**: 오토바이 배달, 배달 박스, 음식 앱 아이콘 등 한국 문화 요소

### 모바일 가시성 규칙

유튜브 모바일 피드에서 썸네일은 약 160×90px 크기로 축소된다:

- **주목 요소(hero element)는 크고 단순하게**: 실루엣만으로 인식 가능한 단순하고 굵직한 형태 우선
- **요소 수 최소화**: 핵심 오브젝트 1~2개로 제한. 3개 이상은 모바일에서 뭉개짐
- **극단적 대비**: 주목 요소와 배경 사이 명도 차이 극대화
- **윤곽선/광채 효과**: rim light, glow, 강한 edge lighting으로 배경과 확실히 분리
- **색상 대비**: 주목 요소에 채도 높은 포인트 컬러, 배경은 채도 낮은 색

### 장르별 이미지 전략

| 장르 | 유력 구도 | 핵심 오브젝트 | 색채 톤 |
|------|-----------|-------------|---------|
| 경제/시사 | 균열/붕괴, A vs B, 화살표 | 브랜드 로고, 건물, 그래프 | 어두운 배경 + 빨강/노랑 포인트 |
| 야담/역사/스토리 | 인물 감정, Before→After | 인물, 상징물, 장소 | 드라마틱 조명, 따뜻한→차가운 전환 |
| 교육/정보 | Before→After, 화살표/그래프 | 결과물, 도구, 숫자 | 밝은 배경 + 포인트 색 |
| 엔터테인먼트/리뷰 | 인물 감정, 스케일 대비 | 인물 표정, 제품 | 고채도, 밝은 톤 |
| 테크/IT | A vs B, 스케일 대비 | 제품, 앱 화면, 로고 | 깔끔한 배경 + 브랜드 컬러 |

### 3요소 역할 분담 원칙

| 요소 | 역할 | 전달 방식 | 예시 |
|------|------|-----------|------|
| **제목** | 합리적 클릭 이유 | 정보적 약속 (텍스트) | "배민이 무너진 진짜 이유" |
| **썸네일 텍스트** | 감정적 호기심 | 시각적 훅 (짧은 텍스트) | "몰락" |
| **썸네일 이미지** | 시각적 보완 | 감정/상황 전달 (비주얼) | 균열 가는 배민 로고 |

### 겹침 가이드

키워드가 겹치는 건 OK — 오히려 메시지를 보강한다. 완전히 동일한 문장을 복붙하는 것만 피한다.
겹침을 피하려고 썸네일 텍스트를 억지로 줄이거나 빈약하게 만들지 않는다.

### 시너지 체크리스트

1. **메시지 보강**: 키워드 겹침으로 메시지를 강화하되, 동일 문장 복붙은 피하는가?
2. **감정 상승**: 3요소를 합쳤을 때 각각보다 감정 강도가 높아지는가?
3. **호기심 갭 극대화**: 3요소가 합작하여 "알고 싶다"는 충동을 만드는가?
4. **약속 이행 가능**: 제목이 약속한 것을 영상이 실제로 이행할 수 있는가?
5. **모바일 인지**: 160×90px에서 이미지+텍스트가 동시에 인식 가능한가?

## 이미지 차용 방법론

### ★ 레퍼런스 비주얼 적극 차용 ★

**잘 터진 영상의 썸네일 비주얼을 적극적으로 차용한다. 핵심은 "검증된 구도/색채/오브젝트를 우리 주제에 맞게 변형"하는 것이다.**

**차용 원칙**:

1. patterns.md의 "썸네일 비주얼 비교표"에서 각 레퍼런스의 구도/오브젝트/색채를 확인한다
2. **조회수 1위 레퍼런스의 비주얼을 1순위 참고 대상**으로 삼는다
3. 각 비주얼에서 효과적인 요소(구도, 오브젝트, 색채, 감정)를 식별한다
4. 아래 차용 레벨 중 하나를 적용한다

**차용 레벨** (이미지 버전):

| 레벨 | 설명 | 예시 |
|------|------|------|
| **L1: 구도 차용** | 레퍼런스 썸네일의 구도를 거의 그대로 가져오고 핵심 오브젝트만 교체 | A vs B 좌우 분할 구도를 그대로, 로고만 교체 |
| **L2: 요소 차용** | 레퍼런스에서 효과적인 시각 요소(색채, 오브젝트, 감정)를 뽑아 다른 구도에 배치 | 균열 효과만 가져와서 다른 구도에 적용 |
| **L3: 멀티 리믹스** | 2개 이상 레퍼런스에서 시각 요소를 조합 | A의 구도 + B의 색채 |
| **L4: 감정 차용** | 레퍼런스의 감정 톤만 가져오고 비주얼은 완전히 새로 | 같은 위기감, 완전히 다른 이미지 |

**핵심: L1~L2를 두려워하지 않는다.** 잘 터진 비주얼의 구도와 색채를 적극 차용하는 것이 CTR에 유리하다.
