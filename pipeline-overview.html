단계별 상세 과정

각 카드를 클릭하면 자세한 설명을 볼 수 있어요

1

📥 재료 수집

COLLECT — 참고할 영상들의 정보를 긁어와요
요리로 치면 장보기 단계예요.
"이런 영상처럼 만들어줘" 하고 유튜브 링크를 주면,
그 영상들의 제목, 자막, 썸네일, 댓글을 자동으로 가져와요.
🐍
collect.py (자동수집 프로그램)
받는 것 (입력)
유튜브 URL 1개~여러개
만드는 것 (출력)
meta.md transcript.txt thumbnail.webp
영상마다 폴더 1개 (001, 002…)
클릭하면 자세히 →
수집하는 것들:
  • meta.md — 제목, 조회수, 좋아요, 구독자 수, 상위 댓글 10개
  • transcript.txt — 영상의 전체 자막 (나레이션 텍스트)
  • thumbnail.webp — 썸네일 이미지

참고 영상이 3개면 001, 002, 003 폴더가 만들어지고
각각 위 3개 파일이 들어가요.
2

🔍 영상 분석

ANALYZE — 각 영상을 깊이 파헤쳐요
수집한 영상들을 하나하나 왜 잘됐는지 분석해요.
"이 영상은 제목이 이래서 클릭을 유도했고,
처음 5초에 이렇게 시작해서 시청자를 잡았구나" 같은 거요.
🔍
영상분석가 (video-analyst) ⚡ 영상마다 1명, 동시 분석
받는 것 (입력)
meta.md transcript.txt thumbnail.webp
← 1단계에서 수집한 것
만드는 것 (출력)
analysis.md
영상마다 1개씩
클릭하면 자세히 →
7가지를 분석해요:
  • ① 이 영상이 뭘 다루는지 한줄 요약
  • ② 썸네일 & 제목이 왜 클릭을 부르는지
  • ③ 처음 몇 초(훅)가 어떻게 시선을 잡는지
  • ④ 본문을 어떤 순서로 풀어가는지
  • ⑤ 어떤 감정을 자극하는지 (놀라움? 분노? 궁금?)
  • ⑥ 댓글 반응 — 시청자들이 어디에 열광하는지
  • ⑦ 제목이 약속한 것 vs 실제 내용

영상 3개면 3명이 동시에 분석해서 시간을 아껴요.
3

🧩 패턴 추출 & 팩트체크

DATA PREP — 공통점을 찾고, 숫자를 검증해요
두 명이 동시에 일해요.
한 명은 "잘되는 영상들의 공통 패턴"을 뽑고,
다른 한 명은 영상에 나온 숫자·사실을 인터넷으로 검증해요.
⚡ 이 두 명이 동시에 일해요
🧩
패턴분석가 (pattern-extractor)
받는 것 (입력)
analysis.md ×N개 meta.md ×N개
← 2단계의 분석 결과들
만드는 것 (출력)
patterns.md
잘되는 영상의 공통 공식
🔬
데이터조사관 (data-researcher) 🌐 웹검색 가능
받는 것 (입력)
transcript.txt ×N개 analysis.md ×N개
← 영상 자막 + 분석 결과
만드는 것 (출력)
factcheck.md verified-data.md
팩트체크 전체 기록 + 검증된 것만
클릭하면 자세히 →
패턴분석가가 하는 일:
  • 모든 분석을 비교해서 "잘되는 영상의 공통 공식" 추출
  • 예: "잘되는 영상은 첫 문장에 결과를 먼저 보여줌"
  • 썸네일/제목 패턴, 감정 흐름, 구조 패턴 등

데이터조사관이 하는 일:
  • 영상에 나온 숫자/통계를 웹에서 직접 검색해서 확인
  • ✅ 맞음 / ❌ 틀림 / ❓ 불확실 로 분류
  • 검증된 데이터만 따로 모아서 대본에 쓸 수 있게 정리
  • + 추가 리서치: 최신 뉴스, 놓친 각도 등도 조사
4

🎯 크리에이티브 전략

STRATEGY — 어떤 영상을 만들지 결정해요
여기가 가장 중요한 단계예요.
"어떤 각도로 이야기할지, 제목은 뭘로 할지,
썸네일은 어떻게 만들지, 첫 5초는 뭘로 시작할지"
— 영상의 전체 방향을 여기서 정해요.
🎯
전략가 (strategist)
받는 것 (입력)
patterns.md verified-data.md 채널 profile.md
← 3단계의 패턴 + 검증 데이터
만드는 것 (출력)
concept.md hook-intro.md prompts.json
컨셉 + 훅&인트로 + 썸네일 프롬프트
클릭하면 자세히 →
전략가가 만드는 것:
  • 3개의 완전히 다른 컨셉을 먼저 만들어요
  • 각 컨셉마다: 관점 + 핵심약속 + 제목 후보 + 썸네일 텍스트 + 첫 5초
  • 셀프 평가 후 가장 좋은 1개를 선택 (또는 사용자가 고름)
  • 확정된 전략으로 썸네일 이미지 프롬프트 5개도 생성

핵심 원칙:
제목(이성적 클릭 이유) + 썸네일 글자(감정 자극) + 썸네일 이미지(시선 끌기)
이 세 가지가 서로 보완하되, 겹치지 않아야 해요.
5

📐 대본 뼈대 잡기

OUTLINE — 전체 구조와 분량을 설계해요
집 짓기 전에 설계도를 그리는 단계예요.
"파트1은 이런 내용으로 2분, 파트2는 저런 내용으로 3분…"
이렇게 전체 뼈대를 잡아요.
🎬
PD (직접 설계)
받는 것 (입력)
concept.md hook-intro.md patterns.md verified-data.md
← 3~4단계의 패턴 + 전략 + 검증 데이터
만드는 것 (출력)
outline.md
파트별 분량 + 내용 설계도
클릭하면 자세히 →
outline.md에 들어가는 것:
  • 전체 영상 길이 & 파트별 목표 분량 (글자 수)
  • 파트별 핵심 내용 & 전달할 감정
  • 어디서 시청자를 잡아둘지 (이탈 방지 장치)
  • 용어 매핑 (어려운 말 → 쉬운 말)
  • 9가지 자체 점검 항목 체크

중요: 제목이 "3가지 이유"라고 했으면
반드시 3개 파트가 있어야 해요. 제목이 구조를 결정해요.
6

✍️ 대본 집필

DRAFT — 실제 나레이션을 써요
드디어 실제 대본을 쓰는 단계!
파트가 3개면 작가 3명이 동시에 각자 파트를 써요.
다 쓰면 하나로 합치고, 분량이 맞는지 검사해요.
✍️
작가 (script-writer) ⚡ 파트마다 1명, 동시 집필
받는 것 (입력)
outline.md (자기 파트) concept.md verified-data.md
← 5단계 뼈대 + 전략 + 데이터
만드는 것 (출력)
_draft_part1.md _draft_part2.md …
파트별 나레이션 원고
🐍
merge_draft.py + validate_draft.py (합치기 & 검사)
받는 것 (입력)
hook-intro.md _draft_part*.md
← 훅&인트로 + 파트별 원고
만드는 것 (출력)
draft.md (합본)
분량 90% 미만 → 재작성 1회
클릭하면 자세히 →
작가의 규칙:
  • outline.md에서 정한 분량 꼭 맞추기
  • 검증된 데이터(verified-data.md)만 사용
  • 쉬운 말로 쓰기, 한 문장에 숫자 3개 이하
  • 채널 톤에 맞추기

분량 검사:
목표 대비 90% 미만이면 ❌ 불합격 → 다시 쓰기 (1회만)
7

✅ 품질 검수

REVIEW — 대본의 완성도를 검증해요
출판 전 편집자가 원고를 검토하는 것처럼,
약속한 내용이 다 들어갔는지, 틀린 정보는 없는지,
새로 추가된 내용은 사실인지까지 꼼꼼히 확인해요.
✅
검수관 (script-reviewer) 🌐 웹검색 가능
받는 것 (입력)
draft.md outline.md concept.md verified-data.md
← 대본 + 기준이 되는 문서들
만드는 것 (출력)
review.md
통과 or 수정 판정
클릭하면 자세히 →
5가지 체크리스트:
  • ① 핵심 약속 — 제목에서 약속한 걸 대본이 지키는지
  • ② 구조 — 뼈대(outline) 대로 쓰였는지
  • ③ 문체 — 쉬운 말, 적절한 숫자 사용
  • ④ 톤 — 채널 분위기에 맞는지
  • ⑤ 새 주장 — 작가가 새로 넣은 내용을 웹 검색으로 검증

판정:
🟢 통과 → 최종 단계로!
🔴 수정 → 작가에게 돌려보내기 (1회만)
🎉

🏁 완성!

DONE — 대본이 완성됐어요
마크다운 서식을 벗겨내고 순수 나레이션 텍스트만 뽑아요.
이 파일을 영상 제작 사이트에 올리면
TTS(음성합성) → 영상 제작이 자동으로 진행돼요.
🐍
finalize.py (최종 변환 프로그램)
받는 것 (입력)
draft.md
← 검수 통과한 대본
만드는 것 (출력)
script.txt
TTS용 순수 나레이션 텍스트
클릭하면 자세히 →
최종 산출물:
  • script.txt — 순수 나레이션 텍스트 (TTS용)

이전 단계에서 이미 생성된 부가 산출물:
  • youtube.md — 업로드용 제목, 설명, 태그
  • prompts.json — 썸네일 이미지 생성용 프롬프트 5개 (4단계 전략에서 생성)

글자 수와 예상 재생시간(400자/분 기준)도 알려줘요.
