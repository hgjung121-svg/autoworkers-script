# 경제 대본 작성 가이드

> **핵심 원칙: 뉴스를 나열하지 않는다. 뉴스를 해석한다.**

## 0. 심층 대본 3대 원칙 (2026-04 업데이트 — 모든 섹션보다 우선)

### 원칙 1. 해석 70% / 팩트 30%
- 본문 시간·분량 배분: 팩트 압축 30% / 심층 해석 70%
- 지표·수치 나열에 머무르면 포털 뉴스와 차별점 소멸 → 심층 채널은 **"왜·누가·어떻게·어디로"**의 해석 깊이로 승부
- 팩트는 해석의 근거로만 사용. 팩트 자체가 파트를 끌고 가면 실패

### 원칙 2. 필수 심층 레이어 — 최소 3종 포함
한 파트 = 하나의 해석 축. 아래 5종 중 최소 3개를 본문에 반드시 포함.
1. 🏛️ **구조 해부** — 왜 이 구조가 만들어졌나 (역사·제도·이해관계 근원)
2. 🎭 **이해관계자 전략** — 누가 무슨 의도로 움직이나 (행동 → 의도 → 효과)
3. 🪞 **오독 지점** — 시장/뉴스가 지금 잘못 읽고 있는 한 줄
4. 📜 **과거 패턴 비교** — 비슷한 국면에서 어디서 갈라졌나
5. 🔮 **시나리오 분기** — 해석이 달라지는 변수

### 원칙 3. 숫자 하드 리밋
- 한 파트당 숫자 **최대 3개**
- **수치 연속 나열 문장 전면 금지**: "A는 7064, B는 24259, C는 49149" 같은 형태 X
- 모든 숫자 뒤에는 반드시 **"이게 의미하는 건"** 해석 문장이 따라와야 함
- 같은 파트 안에서 같은 카테고리 숫자(지수 3개 연속, 수출액 4개 연속) 금지

## 1. 훅 (첫 5초) — 3패턴 택일

아래 3패턴 중 주제에 맞는 **1개 선택**. "손실+숫자"는 유일한 답이 아니다.

### 패턴 A. 해석형 (숨겨진 한 줄)
- "시장은 X로 읽고 있어. 근데 발표문에 묻힌 한 줄이 진짜야"
- 원문·공식 발표·보고서에 묻힌 결정적 문장을 1초에 투하

### 패턴 B. 구조형 (30년의 결정)
- "이 구조를 만든 건 30년 전 결정 하나야"
- 현재 현상의 뿌리를 한 문장으로 압축

### 패턴 C. 손실+숫자형 (기존)
- 첫 5초 안에 구체 숫자 1개 + 손실/위기 프레이밍
- 단, 사용 시에도 숫자 뒤엔 즉시 해석 문장 (나열형 X)

**공통 규칙:**
- 시청자 **내면 독백을 대신 말한다** — 질문형("알고 있으세요?")보다 독백형("그래서 뭐가 달라지는데?")
- "오늘은 ~에 대해" 식 나열형 도입 금지

## 2. 인트로 (훅 후 20~30초)
- "끝까지 보면 뭘 알게 되는가" 구체적 약속
- **신뢰 근거 즉시 제시** ("한국은행 보고서 36페이지를 직접 뜯어봤는데", "이번 주 증권사 리포트를 5개 비교해봤더니")
- 시청자가 아는 정보 공감 → "그건 빙산의 일각"으로 뒤집기
- 시그니처 "돈의 흐름을 따라가보면" 1회 사용

## 3. CTA (인트로 직후!)
- **훅+인트로 끝나고 본론 진입 전에 배치** (클로징이 아닌 초반!)
- "구독 좋아요 알림설정" 직접 나열 금지 — 영상 주제와 연결해 자연스럽게
- 예: "이런 분석 놓치기 싫으면 구독 한번 눌러두고 시작하자"
- 클로징에는 채널 재방문/다음 영상 예고만

## 4. 본론 구조
### 레이어 파고들기 (필수)
```
L1. 표면 뉴스(What) + 생활 번역 → L2. 프레임 전환("진짜 봐야 할 건 따로 있어") → L3. 구조적 원인(Why) → L4. 전략적 심리/역사 → 시나리오 전망
```
- **최소 2회 레이어 전환** (이전 레이어를 무력화하며 진입)
- 인과관계 도미노 (A→B→C→내 지갑)
- 모든 파트에 **글로벌→한국→내 일상** 3단 착지 (2~3분마다)
- 행위자 행동 → **의도 해부** (행동→의도→효과)

### 역설/반전 프레이밍
- 파트 도입마다 상식을 뒤집는 한 문장 배치 ("안전자산이 오히려 위험하다", "금리 인하가 악재")
- 근거 제시 후 "뉴스에서 안 알려주는 / 보고서를 직접 뜯어봤더니" 정보 희소성 프레이밍

### 추가 구조 (주제에 맞게 선택)
- 역추적 분석법 / 양면 교차 해부 / 오해 해체 연쇄 / 불리한 팩트 선제 공개

## 5. 비유/숫자
- 추상 개념 등장 **즉시** 1초 비유 배치 (설명 후 비유 X, 비유가 설명을 대체)
- 감정이 실린 비유 ("양적완화 = 불난 집에 기름 붓는 격")
- 숫자는 반드시 **Before→After 대비 쌍** ("금리 1.75%→3.5%, 월이자 42만→84만 원")
- 한 문장에 숫자 2개 이하
- **섹션 0 원칙 3 재확인**: 파트당 숫자 최대 3개. 연속 나열 금지. 숫자 뒤엔 즉시 해석.

## 6. 리텐션
- 2~3분 간격으로 배치 (전환 멘트/오픈루프/질문형 인터럽트/재반전)
- 파트 전환 직전에 **오픈루프 설치** ("여기까지가 1차 충격이야. 근데 진짜 문제는 다음이거든")
- "그런데 여기서 소름끼치는 게" 영상당 1회만

## 7. 감정 아크
```
공포/박탈감(초반 30%) → 공포-안도 반복 사이클(중반 40%) → 안도+실행 의지(후반 30%)
```
- **파트마다 공포→안도 사이클 반복** (한 이슈 해결 후 새로운 위기 시작 = 롤러코스터)
- 공포만 쌓지 않는다. **클로징은 통제감+실행 가능한 행동**으로 착지
- 감정 스케일링: 개인→시장→국가→다시 개인
- 역사적 전례로 통제감 부여
- 최소 2개 시나리오 + 대응 가이드 + 핵심 변수 체크리스트 3~5개

## 8. 금지사항
1. 과장 프레이밍 금지 (본문에서 귀결 못 하면 신뢰 하락)
2. 제목 약속 반드시 이행
3. 검증 안 된 수치/주장 금지
4. 감정 수식어 남발 금지 ("충격", "경악", "발칵")
5. 뉴스 단순 나열 금지
6. **수치 연속 나열 문장 금지** (섹션 0 원칙 3)
7. **해석 없는 지표 투하 금지** — 숫자 다음 문장이 "이게 의미하는 건"으로 시작하지 않으면 재작성

## 9. 자료 수집 (2026-04 업데이트 — R0~R7)

### R0. 사전 입력 (선택적)

### R1. What (3회, 지표+이슈 속보)

### R2. Who/Why — 원문 최소 5개 WebFetch ★핵심 강화★
- 1차 소스 5개 직접 열람:
  - 정부·기관 공식 발표문 1개

  - 보고서/논문 1개 (CSIS, 산업연구원, 무역협회 등)
- 이해관계자 발언 검색 3회:
  - "{주제} 공식 발표 원문" / "{주제} 관계자 발언" / "{주제} 역사적 전례"

- 오류 발견 시 script.txt 수정 후 재저장

- 24시간 이내 기사만. 원문 WebFetch **5개 필수**.

## 10. 제목/썸네일 설계
- **제목**: 3패턴 택일 — (A) 손실 프레이밍+숫자+정보 희소성 / (B) 해석 반전(숨겨진 한 줄) / (C) 구조 폭로형
- **썸네일**: 인물 중심 + 구체 사건/장소/수치 + 어두운 배경+빨강/노랑 포인트
- **역할 분담**: 썸네일=인물+수치 충격(시각), 제목=해석 예고(텍스트). 같은 정보 반복 금지

### 썸네일 A/B 테스트 승리 공식 (2026-04 실데이터)
유튜브 Studio A/B 테스트 결과에서 검증된 패턴:
- 🏆 **1번(A안) — 인물 + 구체 사건 + 짧은 텍스트** (TSMC 영상 A/B 53.8% 승리, 36시간 영상 43.6% 승리)
- 2번(B안) — 차트/오브젝트 중심 (대조군, 약 27~34% 선호)
- 3번(와일드카드) — 추상 프레이밍 (대조군, 약 12~27% 선호)

**prompts.json 3후보 설계 원칙:**
- 1번은 반드시 **인물(실명 가능) + 구체적 장소·사건·수치** ("카타르 헬륨 산단 드론 타격"처럼 장소+행위+대상)
- 추상 프레이밍("판이 갈린다", "운명의 시간") 단독 썸네일은 3번(와일드카드)에만
- 텍스트는 10자 이하 간결형이 CTR 더 높음

### 대본 분량 기준
